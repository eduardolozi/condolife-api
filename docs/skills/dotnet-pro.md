# Skill: dotnet-pro

## Purpose
Apply robust .NET/C# implementation practices aligned with this codebase.

## Use When
- Implementing use cases, handlers, validators, or controller actions.
- Refactoring code for readability and maintainability.
- Reviewing naming, layering, and DI composition.

## Guardrails
- Follow existing naming and folder patterns.
- Keep methods focused and explicit.
- Prefer constructor injection and explicit dependencies.
- Avoid introducing new abstractions unless repeated pain is proven.

## Quick Checklist
- Code follows existing project style.
- Dependencies are explicit and minimal.
- Error paths are clear and intentional.
- No unnecessary architectural layers added.
